# 프로젝트 소개

한정판 굿즈를 선착순으로 판매하는 타임딜 커머스 플랫폼입니다.

사용자는 정해진 시간에 오픈되는 상품을 구매할 수 있으며, 결제는 외부 PG(Payment Gateway)를 통해 처리됩니다. 단순한 상품 판매 서비스가 아니라 실제 서비스 환경에서 발생할 수 있는 동시성 문제, 외부 결제 시스템 연동, 데이터 정합성 보장 등의 기술적 과제를 해결하는 것을 목표로 합니다.

또한 기업 고객(B2B)을 대상으로 대량 SMS 발송 기능을 제공하여 대규모 요청 처리와 비동기 이벤트 기반 아키텍처를 경험할 수 있도록 설계합니다.

---

# 프로젝트 목표

본 프로젝트의 목표는 단순 CRUD 중심의 커머스 서비스 구현이 아닌, 실무에서 자주 마주치는 핵심 백엔드 문제를 직접 해결하고 설계하는 것입니다.

## 비즈니스 목표

* 한정판 굿즈 타임딜 판매 서비스 구현
* 외부 PG 연동 기반 결제 시스템 구축
* 기업 고객 대상 대량 SMS 발송 기능 제공
* 주문, 결제, 알림 도메인 분리 및 관리

## 기술적 목표

- DDD 기반 도메인 설계 
- 회원 / 상품 / 재고 / 주문 / 결제 도메인 
- Redis Distributed Lock 기반 선착순 재고 차감 
- 주문/결제 상태 머신 
- PG Mock 연동 
- PG 실패 시 재고 복구 + 주문 실패 처리 
- Idempotency Key로 중복 주문/결제 방지 
- Transactional Outbox Pattern 
- RabbitMQ 또는 Redis Stream 기반 SMS 비동기 발송 
- 재시도 / DLQ / 발송 상태 추적 
- 동시성 테스트 
- k6 또는 JMeter 부하 테스트

# 핵심 문제 정의

## 1. 선착순 판매 환경에서의 동시성 문제

타임딜 상품은 짧은 시간 동안 많은 사용자가 동시에 구매를 시도합니다.

재고가 1개 남은 상황에서 여러 사용자가 동시에 결제를 요청할 경우 재고가 음수로 감소하거나 중복 판매가 발생할 수 있습니다.

이를 해결하기 위해 Redis Distributed Lock을 적용하여 재고 차감 과정의 원자성을 보장합니다.

---

## 2. 외부 PG 결제 실패 시 데이터 정합성 문제

사용자가 상품 구매를 시도하면 재고 확보와 결제 승인 과정이 함께 이루어집니다.

하지만 외부 PG 장애, 네트워크 오류, 사용자 결제 취소 등의 이유로 결제가 실패할 수 있습니다.

이 과정에서 이미 확보된 재고가 정상적으로 복구되지 않으면 실제 재고와 시스템 재고 간의 불일치가 발생하게 됩니다.

이를 해결하기 위해 보상 트랜잭션(Compensation Transaction)을 적용하여 결제 실패 시 주문 취소 및 재고 복구를 수행합니다.

---

## 3. 대량 SMS 발송 처리 문제

기업 고객은 수천 건 이상의 SMS 발송을 요청할 수 있습니다.

모든 요청을 실시간으로 처리할 경우 외부 SMS API의 TPS 제한에 걸리거나 서비스 전체 성능 저하가 발생할 수 있습니다.

이를 해결하기 위해 메시지 큐 기반 비동기 처리 구조를 도입하고, 실패 재시도 및 발송 상태 추적 기능을 구현합니다.

---

## 4. 외부 시스템 장애 전파 문제

결제 시스템과 SMS 시스템은 모두 외부 서비스에 의존합니다.

외부 서비스의 응답 지연 또는 장애가 발생할 경우 전체 서비스의 응답 속도가 저하되거나 장애가 전파될 수 있습니다.

이를 해결하기 위해 도메인 이벤트 기반 비동기 처리 구조를 적용하여 주문, 결제, 알림 도메인의 결합도를 낮추고 장애 격리를 수행합니다.
